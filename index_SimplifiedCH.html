哈啰，欢迎到访！此网页为“建筑能量模拟工作流程自动化（建筑能量模拟工作流程自动化）”的问卷调查结果。黄安蕾与陈欣于2017年6月中旬进行了问卷调查，于2017年建筑模拟大会（Building Simulation 2017 Conference）上报调查结果，同时邀请两位嘉宾Peter Ellis @ Big Ladder Software 和Weili Xu @ Buildsimhub 进行座谈。 你可能会好奇这个问卷的内容及标题是如何产生的。在过去两年半身为建筑能源分析师的日子里，大部分时间将时间，心力花在琐碎且重复的工作内容（也就是大家熟悉的复制和贴上）。我想知道是否有任何技术或工具可以改善工作流程甚至解决问题。我开始学习写的Python帮我做简单的分析及抓取文字和数字，重复及琐碎的手动工作项目减少了，而我有更多的时间来处理并解决更复杂的问题。去年（2016年），参加了ASHRAE建筑绩效分析大会（ASHRAE Building Performance Analysis Conference），有幸聆听Michael O'Keefe和Peter Eilis的向软件工程师学习如何减化能源模拟过程（Model Like a Programmer）的演讲。在演讲中，Michael讲述了建模者（建筑能源建模师/分析师）和软体工程师（程序员）之间的相似之处，并展示了将写程式的技能应用到工作流中的优点。从这演讲中我受益许多。尔的研究搜索中发现已经有许多既有的工具和程式范本，我们行业中的许多先驱者都在开发工具来帮助我们减少这些迭代工作流程。那么下面的问题呢，我相信大多数人一直在想同样的问题，我们大多数人在我们的工作流中使用这些工具？我们大部分人是否准备好我们的能力，如编程技能？我们的公司和社群是否准备好迎接趋势并有资源支持着我们（如培训或研讨会）？为了回答这些问题，我们需要这个建筑模拟社群的先进后辈们的帮助，这份调查也就此诞生。很幸运的，我们收到了超过70份的回覆。再次感谢帮助我们完成此调查的人。

在调查中，我们将问题分为四个类别：。自动化，编程，个人发展和公司支持自动化问题是为了了解不同建模过程中当前的工具采用率脚本和编程的问题是要知道人们正在使用哪些编程语言来编写脚本，以及它们与他们做什么？这个雷达图显示了我们行业目前使用和计划学习的专业的编程语言。个人发展问题正在考察学习规划的障碍和优势。最后一个类别是公司支持，这是为了知道这些具体技能是否由公司支持。

受访者背景资料

我们总共收到73份回覆，超过50％的参与者位于美国，其余的来自欧洲，南美洲，大洋洲和亚洲。在职业方面，绝大多数是能源建模者（energy modeler）或能源分析师（energy analyst)），其次是学术专业人士与建筑师。
